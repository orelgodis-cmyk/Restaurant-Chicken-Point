# Chicken Point Management V4.2

גרסה מאוחדת עם עדכון שעות עובדים גם ביום שלמחרת.

## חדש בגרסה 4.2
- אפשר לשמור דוח יומי עם שעת יציאה מתוכננת בלבד.
- בדוח ההיסטורי מופיע כפתור **עדכן שעות עובדים**.
- ניתן להוסיף למחרת את שעת היציאה בפועל ואת סיבת החריגה.
- החריגה מחושבת מחדש אוטומטית.
- כל עדכון נשמר עם תאריך ושעה, והגרסה הקודמת של שעות העובדים נשמרת בהיסטוריית הדוח.
- דוחות העובדים והדוח השבועי מתעדכנים לפי השעות החדשות.

# Chicken Point Management V4.1 — גרסה מאוחדת

מערכת ניהול יומית למסעדה עם Firebase Firestore, שמאחדת את מערכת הניהול החדשה עם כל הכלים מהגרסה הקודמת.

## ארבעת האזורים הראשיים
1. היום שלי — משימות, פתיחת יום, טעימות, כמויות, עובדים, אמצע יום וסוף יום.
2. דיווח בעיה — כמות, סיבה, פתרון, תמונה ובדיקה חוזרת.
3. מעקבים פתוחים — עדכון תוצאה וסגירת מעקב עם היסטוריה.
4. דוח שבועי — נתונים אוטומטיים, תובנות והחלטות לשבוע הבא.

## מה חזר מהגרסה הקודמת
- משימות יומיות, שבועיות, חודשיות ותזכורות אוראל — הוספה, ביצוע, פתיחה מחדש ומחיקה.
- היסטוריית ביצוע משימות עם תאריך ושעה.
- איפוס אוטומטי למשימות חוזרות לפי יום, שבוע או חודש.
- דוח אוכל מפורט וסטטיסטיקת אוכל.
- שיפורים והתייעלות.
- היסטוריית דוחות אוכל ודיווחי עובדים ישנים.

## כלים חדשים
- דוח יומי ניהולי מלא.
- מעקב בעיות פתוחות וסגורות.
- שעות יציאה מתוכננות מול בפועל.
- העתקת סיכום יומי מוכן לוואטסאפ לאלעד.
- דוח שבועי הכולל פחת, שעות, בעיות ומשימות שבוצעו.

## שמירת נתונים
הגרסה משתמשת באותו פרויקט Firebase ובאותם אוספים ישנים, ולכן הנתונים הקיימים נשארים זמינים. נוסף אוסף `taskHistory` עבור היסטוריית ביצוע משימות.


## שינויי V4.1
- הוסר תכנון הכמויות היומי ממסך היום שלי כדי לקצר ולפשט.
- מעקב שעות העובדים הועבר לתוך דיווח סוף היום.
- הסיכום היומי לאלעד עודכן בהתאם.
